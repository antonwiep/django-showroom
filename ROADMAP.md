# Roadmap

## Phase 0: Foundation

- Publish repository and governance docs.
- Stabilize contract definitions.
- Establish CI for lint/test/typecheck/contracts.

## Phase 1: Contract hardening

- Finalize `component.json` and `stories.json` schemas.
- Add strict validators and error messages.
- Add compatibility policy for contract evolution.

## Phase 2: Adapter baseline

- Django template adapter parity.
- HTML adapter parity.
- Cotton adapter parity.
- Alpine and vanilla JS behavior parity.

## Phase 3: Split view maturity

- Stable shared args synchronization.
- Side-by-side zoom parity.
- Interaction parity checks (focus/portal/overlay behavior).

## Phase 4: React adapter

- React adapter implementation against existing contract.
- Story parity matrix for core components.
- Performance validation for split mode.

## Phase 5: Ecosystem and adoption

- Starter examples.
- Migration guides for existing teams.
- Documentation polish and external contributor ramp-up.

## Success metrics

- % of components with parity across adapters.
- % of stories with deterministic controls behavior.
- Time-to-first-contribution for new contributors.
- Open issue median age.
