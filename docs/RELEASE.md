# Release Process

## Versioning

Use semantic versioning.

- `MAJOR`: breaking contract changes.
- `MINOR`: backward-compatible features.
- `PATCH`: backward-compatible fixes.

## Pre-release checklist

1. Contract validation passes.
2. Adapter parity suite passes.
3. Split view regression suite passes.
4. Changelog updated.
5. Migration notes added for breaking changes.

## Changelog sections

- Added
- Changed
- Deprecated
- Removed
- Fixed
- Security

## Release cadence

- Weekly or bi-weekly for pre-1.0.
- Predictable schedule once contributor load stabilizes.
